# MapGuide 模板资源说明

> 本文档说明 `SourceCode/data/templates/` 目录下各模板文件的来源、作用、结构，以及专业技术人员如何进行修改维护。
> **范围**：WMS、WFS、WCS、WMTS、TMS 多服务支持

---

## 1. 目录总览

```
SourceCode/data/templates/
├── mapfile-schema-8-4.json    ← 官方 JSON Schema（唯一外部来源）
├── aliases.json                ← 业务别名映射
├── required.json               ← 必填/条件必填规则（含服务类型条件）
├── phase-map.json              ← 阶段归属映射
├── defaults-override.json      ← 默认值覆盖/补充
├── dependencies.json           ← 跨字段依赖关系（含服务类型条件）
├── custom-allowed.json         ← 自定义属性白名单
├── service-metadata.json       ← 服务类型 → METADATA 参数白名单映射（新增）
└── object-fields.json          ← Schema 未覆盖的业务补充字段
```

**使用方式**：运行 `SourceCode/scripts/generate_rules.py` 将上述文件合并为 `SourceCode/data/mapguide_rules.json`，供运行时 `TemplateMapper` 唯一读取。

## 2. 数据来源与可靠性证明

### 2.1 mapfile-schema-8-4.json — MapServer 官方 JSON Schema

| 属性 | 说明 |
|------|------|
| **来源** | MapServer 官方 GitHub 仓库: `mapserver/mapserver` → `mappyfile` 项目 |
| **版本** | MapServer 8.4 |
| **格式** | JSON Schema Draft 2020-12 |
| **获取方式** | 官方 mappyfile Python 库内置，或直接从 [mapserver/mapserver](https://github.com/MapServer/MapServer) 仓库提取 |
| **可靠性证明** | ① mappyfile 是 MapServer 官方推荐的 Mapfile 解析/生成库；② JSON Schema 由官方维护，与 MapServer 源码关键字定义同步；③ `mappyfile.validate()` 直接使用此 Schema 进行语法校验 |

**Schema 内容**：定义了 MAP / LAYER / CLASS / STYLE / LABEL / WEB / METADATA 等对象类型的所有合法字段、数据类型、枚举值、数值范围、默认值等。

> **验证 Schema 可靠性**：在 Python 中执行 `mappyfile.validate(mf_dict, version=8.4)`，其内部加载的 Schema 与本文档中的 `mapfile-schema-8-4.json` 一致。

### 2.2 业务覆盖文件（人工维护，8 个 JSON 文件）

这 **9 个**文件是 MapGuide 项目根据**实际业务场景**对官方 Schema 的**补充和覆盖**，全部人工编写和维护。它们不替代 Schema，而是在合并时叠加在 Schema 之上。

## 3. 各模板文件详解

### 3.1 aliases.json — 业务别名映射

**作用**：将用户的自然语言/中文输入转换为 Mapfile 标准参数值。

**适用场景**：
- 用户在 ConfigTree 中输入 `"点"` → 后端自动转换为 `"point"`
- LLM 返回 `"红色"` → 后端自动转换为 `[255, 0, 0]`
- 中文操作界面中的选项值映射

**结构**：
```json
{
  "<对象类型>": {
    "<字段名>": {
      "<别名>": "<标准值>"
    }
  }
}
```

**当前支持的对象类型**：MAP / LAYER / STYLE / CACHE

**修改示例**（添加更多颜色别名）：
```json
"STYLE": {
  "color": {
    "紫": [128, 0, 128],
    "紫色": [128, 0, 128]
  }
}
```

> ⚠️ **注意**：别名不改变合法值域。`enum` 的合法值仍以 Schema 定义为准。别名只是多入口映射到同一个标准值。

---

### 3.2 required.json — 必填规则

**作用**：定义各对象类型的必填字段、业务必填字段、条件必填规则。

**三个层级**（经 MapServer 8.6.3 源码验证修正）：

| 层级 | 说明 | UI 标记 | 验证时机 | 示例 |
|------|------|---------|---------|------|
| `required` | **语法绝对必填**。不填则 MapServer 解析时直接报错（`msSetError` → `MS_FAILURE`）。Mapfile 无法加载。 | 红色 `*` | L4 mappyfile | `LAYER.type` |
| `business_required` | **业务建议必填，但有默认值**。不填时系统已自动填充默认值，功能可正常工作。不应标红 `*`。 | 灰色或隐藏 | 提示但不阻塞 | `METADATA.ows_title`（服务启用时） |
| `required_when` | **条件必填**。满足特定条件时不填会产生服务级警告（`OWS_WARN`）或阻塞渲染。 | 红色 `*`（条件触发时） | L3 semantic | `connection` 在 `connectiontype in ['postgis','ogr','wms']` 时必填 |

**关键区分原则**：

- **`required` ≠ `business_required`**：`business_required` 中的字段如果已有 `defaults-override.json` 或 Schema 默认值，则**不应再标记为必填**。例如 `MAP.imagetype`（默认 `png`）、`LAYER.status`（默认 `on`）、`METADATA.ows_enable_request`（默认 `*`）都不应出现在 `business_required` 中。
- **`required` 的来源是 MapServer 源码**：不是主观判断。只有源码中 `loadXXX()` 函数在 `END` 处明确检查的字段（如 `loadLayer` 检查 `layer->type == -1`）才属于 `required`。
- **`required_when` 的来源是 OGC 服务规范**：WMS/WFS/WCS 的 GetCapabilities 中标记为 `OWS_WARN` 的字段（如 `title`、`srs`）属于条件必填。

**MapServer 源码验证摘录**：

```c
// mapfile.c:4544 — LAYER.type 是语法绝对必填
if ((int)layer->type == -1) {
    msSetError(MS_MISCERR, "Layer type not set.", "loadLayer()");
    return (-1);
}

// mapwms.cpp:3697 — WMS 服务中 title 是服务级必填
/* Layer Name is optional but title is mandatory. */

// mapwms.cpp:2383 — PROJECTION 是条件功能必填
/* If map has no proj then every layer MUST have one or produce a warning */
```

**条件表达式语法**：使用 Python 表达式，支持 `parent.<field>` 引用父对象字段、`value` 指代当前字段值、`session` 指代当前会话（含 `service_types`）。

**服务类型条件示例**：
```json
"METADATA": {
  "required": [],
  "business_required": [],
  "required_when": {
    "ows_title": "len(session.service_types) > 0",
    "wms_title": "'wms' in session.service_types and not map.web.metadata.get('ows_title')",
    "wfs_title": "'wfs' in session.service_types and not map.web.metadata.get('ows_title')",
    "wcs_title": "'wcs' in session.service_types and not map.web.metadata.get('ows_title')"
  }
}
```

> **修正说明**：`ows_title` / `ows_onlineresource` 不应放在 `business_required` 中作为无条件必填，因为：① 不启用任何 OGC 服务时它们完全不需要；② `ows_onlineresource` 可由 `MAP.name` 自动推导（mapwms.cpp:3433）；③ `ows_enable_request` 已有默认值 `*`。

**修改示例**（添加新条件必填）：
```json
"LAYER": {
  "required": ["type"],
  "business_required": [],
  "required_when": {
    "connection": "connectiontype in ['postgis', 'ogr', 'wms']",
    "data": "connectiontype == 'local'",
    "name": "len(session.service_types) > 0"
  }
}
```

> **注意**：`LAYER.name` 不是语法必填（MapServer 解析允许无 name），但启用 OGC 服务时至少一个 LAYER 需要有 name 才能被服务引用。`MAP.name` 同样不是语法必填。

> **MVP 阶段**：服务类型条件校验可暂不实现硬阻断，只作为 LLM 建议依据。

### 3.3 phase-map.json — 阶段归属映射

**作用**：将对象类型映射到 UI 阶段，用于颜色徽章和 Prompt 上下文分类。

**当前映射**：

| 对象类型 | 阶段 | 颜色 |
|----------|------|------|
| MAP / WEB / METADATA | `service` | 绿色 `#16a34a` |
| LAYER | `datasource` | 蓝色 `#2563eb` |
| CLASS / STYLE / LABEL | `style` | 橙色 `#ea580c` |
| CACHE | `cache` | 紫色 `#9333ea` |

**设计意图**：`phase` 是**分类标签**，不是流程驱动器。UI 中每个节点根据所属对象类型显示对应颜色徽章，LLM Prompt 中用于组织参数上下文。

**修改**：如需增加新对象类型，在此文件中添加映射即可。

---

### 3.4 defaults-override.json — 默认值覆盖/补充

**作用**：覆盖或补充官方 Schema 中的默认值，提供业务友好的初始值。

**优先级**：`defaults-override.json` > Schema default > 无默认值

**特殊结构 — 条件默认值**（`STYLE` 对象）：
```json
"STYLE": {
  "conditional": {
    "parent.type == 'point'": {
      "color": [128, 128, 128],
      "symbol": "circle",
      "size": 8
    },
    "parent.type == 'line'": {
      "color": [128, 128, 128],
      "width": 1
    }
  }
}
```

当创建新的 STYLE 节点时，根据父 CLASS 的 `type` 自动填充不同的默认值。

**服务通用前缀默认值**：
```json
"METADATA": {
  "ows_enable_request": "*",
  "wms_enable_request": "*",
  "wfs_enable_request": "*",
  "wcs_enable_request": "*"
}
```

**修改示例**：
```json
"MAP": {
  "imagetype": "png",      // 覆盖 Schema 默认值
  "status": "on",          // 补充 Schema 未定义的默认值
  "maxsize": 4096          // 新增默认值
}
```

### 3.5 dependencies.json — 跨字段依赖关系

**作用**：定义字段间的逻辑依赖，用于驱动验证和 UI 行为。

**关系类型**（`relation`）：

| 关系 | 说明 |
|------|------|
| `requires_when` | 条件要求：当条件满足时，目标字段变为必填 |
| `forbids_when` | 条件禁止：当条件满足时，目标字段应被忽略/隐藏 |
| `derives` | 推导关系：源字段决定目标字段的值（自动计算，不可编辑） |
| `invalidates` | 失效关系：源字段变化时，目标字段的值可能不再有效 |

**结构**：
```json
{
  "source": "LAYER.type",
  "target": "STYLE.symbol",
  "relation": "requires_when",
  "condition": "value == 'point'",
  "description": "point 类型需要 symbol"
}
```

**服务类型相关依赖示例**：
```json
{
  "source": "session.service_types",
  "target": "LAYER.gml_include_items",
  "relation": "requires_when",
  "condition": "'wfs' in value",
  "description": "启用 WFS 时，建议设置 gml_include_items"
}
```

**当前边列表**：
- `LAYER.type` → `STYLE.symbol/size/width/outlinecolor`（4 条 requires_when）
- `LAYER.type` → `STYLE.color/symbol/size/width/outlinecolor`（5 条 derives）
- `LAYER.connectiontype` → `LAYER.connection`（requires_when）
- `LAYER.connectiontype` → `LAYER.data`（forbids_when）
- `LAYER.projection` → `CACHE.grid`（derives）
- `MAP.name` → `METADATA.wms_onlineresource`（derives）
- `session.service_types` → `LAYER.gml_include_items`（requires_when，WFS）
- `session.service_types` → `LAYER.gml_featureid`（requires_when，WFS）
- `session.service_types` → `LAYER.type`（conditional_value，WCS→raster）
- `METADATA.ows_title` → `METADATA.wms_title`（forbids_when）
- `METADATA.ows_title` → `METADATA.wfs_title`（forbids_when）
- `METADATA.ows_title` → `METADATA.wcs_title`（forbids_when）

**修改示例**（添加新依赖）：
```json
{
  "source": "LAYER.type",
  "target": "STYLE.gap",
  "relation": "requires_when",
  "condition": "value == 'line'",
  "description": "line 类型支持虚线间隙"
}
```

### 3.6 custom-allowed.json — 自定义属性白名单

**作用**：控制哪些对象类型允许用户添加 Schema 未定义的自定义属性。

**当前配置**：

| 对象类型 | 允许自定义？ |
|----------|-------------|
| MAP / WEB / METADATA / LAYER / CLASS / CACHE | ✅ true |
| STYLE | ❌ false |

**设计理由**：Mapfile 的 MAP、LAYER 等对象天然支持扩展属性（如自定义 CGI 变量、WMS 扩展参数）。STYLE 对象字段严格由 MapServer 定义，不应允许自定义。

### 3.8 object-fields.json — Schema 未覆盖的业务补充字段

**作用**：为 Mapfile Schema 未定义但业务需要的对象类型补充字段定义。

**典型场景**：
- **METADATA**：开放键值容器，Schema 只定义了 `"type": "object"`，未枚举具体键名。我们补充了 OGC WMS/WFS/WCS 规范字段。
- **CACHE**：MapCache 配置不属于 Mapfile Schema，完全由本文件定义。

**结构**（与 Schema 字段格式兼容）：
```json
{
  "<对象类型>": {
    "<字段名>": {
      "value_type": "string|integer|float|boolean|enum|color|expression",
      "enum": ["可选值列表"],
```

### 5.3 场景三：调整必填规则

**修改文件**：`required.json`

**修改原则**：
1. `required` 只放**语法绝对必填**（MapServer 解析时会报错的字段）
2. `business_required` 只放**没有默认值且业务上必须用户显式填写**的字段
3. `required_when` 放**条件触发时必填**的字段（服务类型条件、字段依赖条件）
4. 已有默认值的字段**不应**出现在 `required` 或 `business_required` 中

**错误示例**（当前配置中存在的问题）：
```json
// ❌ 错误：这些字段都有默认值，不应标记为 business_required
"MAP": {
  "business_required": ["imagetype", "status"]  // imagetype=png, status=on
},
"LAYER": {
  "business_required": ["connectiontype", "projection", "status"]
  // connectiontype=local, projection=["init=epsg:3857"], status=on
},
"METADATA": {
  "business_required": ["ows_title", "ows_onlineresource", "ows_enable_request"]
  // ows_enable_request="*"（默认值），ows_onlineresource 可由 MAP.name 推导
}
```

**正确示例**：
```json
// ✅ 正确：三层语义清晰分离
"MAP": {
  "required": [],
  "business_required": [],
  "required_when": {}
},
"LAYER": {
  "required": ["type"],  // 语法绝对必填：不填 MapServer 报错 "Layer type not set"
  "business_required": [],
  "required_when": {
    "connection": "connectiontype in ['postgis', 'ogr', 'wms']",
    "data": "connectiontype == 'local'",
    "name": "len(session.service_types) > 0"
  }
},
"METADATA": {
  "required": [],
  "business_required": [],
  "required_when": {
    "ows_title": "len(session.service_types) > 0",
    "ows_onlineresource": "len(session.service_types) > 0",
    "wms_title": "'wms' in session.service_types and not map.web.metadata.get('ows_title')",
    "wms_srs": "'wms' in session.service_types and not map.get('projection')"
  }
}
```

**服务类型条件必填示例**：
```json
"METADATA": {
  "required_when": {
    "wms_title": "'wms' in session.service_types",
    "wfs_title": "'wfs' in session.service_types"
  }
}
```

> MVP 阶段条件必填可仅作为提示，不阻断导出。

## 6. 注意事项

1. **`_description` 字段**：每个 JSON 文件顶部可包含 `"_description"` 键，用于说明文件用途。生成脚本会自动移除它，不会进入产物。

2. **Schema 更新**：如需升级 MapServer 版本（如 8.4 → 8.6），替换 `mapfile-schema-8-4.json` 为对应版本的新 Schema，然后重新生成。业务覆盖文件通常不需要改动。

3. **避免重复定义**：`object-fields.json` 中的字段名不应与 `mapfile-schema-8-4.json` 中已定义的字段冲突。如有冲突，以 `object-fields.json` 为准，但这通常意味着设计上有问题。

4. **条件表达式安全**：`required.json` 和 `dependencies.json` 中的条件表达式会被 `eval()` 执行（在受控上下文中）。只应使用简单的比较和成员判断，避免复杂逻辑。

5. **嵌套对象字段的 `editable` 标记**：`generate_rules.py` 会自动检测嵌套对象/数组字段（内部有子字段但未被 `SCHEMA_LOCATIONS` 展开的），并将其 `editable` 设为 `false`。前端使用 `ObjectViewer` 组件渲染为**可折叠 JSON 文本**（默认收起，`editable=false` 时只读）。当前被标记为 `editable: false` 的字段包括：
   - **MAP**：`config`, `legend`, `outputformats`, `querymap`, `reference`, `scalebar`, `symbols`
   - **LAYER**：`composites`, `features`, `grid`, `joins`, `scaletokens`
   - **LABEL**：`styles`

6. **LLM Prompt 过滤**：`TemplateMapper.get_llm_context_summary()` 生成 LLM 参数摘要时，会自动过滤 `editable=false` 的字段，避免 LLM 误建议修改不可编辑内容。

7. **Git 管理**：`data/templates/` 下的所有文件和 `scripts/generate_rules.py` 都应纳入版本控制。`data/mapguide_rules.json` 是生成产物，可选择性纳入版本控制或让 CI 自动生成。

8. **服务类型字段保留**：取消勾选某个服务后，该服务对应的 METADATA 字段值**仍保留在 params 中**，只是 UI 上隐藏。重新勾选后恢复显示。这保证了用户切换服务类型时不会丢失已填写的配置。

### B.2 `msOWSLookupMetadata` 前缀系统

MapServer 通过 `msOWSLookupMetadata(metadata, prefixes, key)` 函数查找 METADATA 参数。`prefixes` 参数决定查找顺序：

| 源码前缀 | 查找顺序 | 说明 |
|---------|---------|------|
| `"M"` | `wms_*` → `ows_*` | WMS 专用 → OWS 通用回退 |
| `"MO"` | `wms_*` + `ows_*` | 同时尝试两组前缀 |
| `"F"` | `wfs_*` → `ows_*` | WFS 专用 → OWS 通用回退 |
| `"FO"` | `wfs_*` + `ows_*` | 同时尝试两组前缀 |
| `"G"` | `gml_*` | GML 专用 |
| `"OFG"` | `ows_*` → `wfs_*` → `gml_*` | OWS → WFS → GML 三级回退 |
| `"C"` | `wcs_*` → `ows_*` | WCS 专用 → OWS 通用回退 |
| `"CO"` | `wcs_*` + `ows_*` | 同时尝试两组前缀 |

**示例**：`msOWSLookupMetadata(&(lp->metadata), "OFG", "featureid")` 会依次查找 `gml_featureid`、`wfs_featureid`、`ows_featureid`。

| **WMTS** | MapCache (`mapcache.xml`) | `<service type="wmts">` | 底层回源到 MapServer WMS |
| **TMS** | MapCache (`mapcache.xml`) | `<service type="tms">` | 底层回源到 MapServer WMS |

**通用前缀回退**：MapServer 在解析 METADATA 时，如果找不到服务专用前缀（如 `wms_title`），会回退到 `ows_title`。本应用不实现这一回退逻辑——后端直接按用户填写的值写入 METADATA，MapServer 在运行时自行处理。

## 附录 C：必填项语义探究结论（2026-06-08）

### C.1 问题背景

用户反馈：当前"仅必填"模式下显示的必填项太少，或者定义不明确——只填必填项后，QA 仍会提示填写其他参数才能导出可用的 Mapfile。

### C.2 根本原因

**`business_required` 与 `required` 在 UI 上无区分**，且 `business_required` 中**混入了大量已有默认值的字段**，同时**真正缺失的关键字段**（如 `MAP.extent`）未被标记。

### C.3 三层语义定义（经 MapServer 8.6.3 源码验证）

| 层级 | 定义 | 验证标准 | 当前问题 |
|------|------|---------|---------|
| **`required`** | 语法绝对必填。不填则 MapServer `loadXXX()` 在 `END` 处返回 `MS_FAILURE`，Mapfile 无法解析。 | 源码中有明确的 `msSetError` + `return (-1)` 检查 | `LAYER.name`、`MAP.name` 被错误标记——它们不是语法必填 |
| **`business_required`** | 没有默认值且业务上必须用户显式填写。不填时系统**不会**自动填充，功能受限。 | 无默认值 + 不填时服务产生 `OWS_WARN` 或功能异常 | 混入了 `imagetype`、`status`、`connectiontype`、`projection`、`ows_enable_request` 等已有默认值的字段 |
| **`required_when`** | 条件必填。满足特定条件时不填会产生服务级警告或阻塞。 | `dependencies.json` 或源码中的条件检查 | 基本合理，但 `METADATA.ows_title` 应从 `business_required` 移入此处 |

### C.4 源码验证要点

| 字段 | 验证位置 | 结论 |
|------|---------|------|
| `LAYER.type` | [mapfile.c:4544](Document/Resources/mapserver-8.6.3/src/mapfile.c#L4544) | ✅ 语法绝对必填：`"Layer type not set."` |
| `LAYER.name` | [mapwms.cpp:3697](Document/Resources/mapserver-8.6.3/src/mapwms.cpp#L3697) | ❌ 不是语法必填： `"Layer Name is optional but title is mandatory."` |
| `MAP.name` | loadMapInternal 无 END 检查 | ❌ 不是语法必填，但用于推导 `onlineresource` |
| `METADATA.title` | [mapwms.cpp:3422](Document/Resources/mapserver-8.6.3/src/mapwms.cpp#L3422) | ⚠️ 服务级必填（`OWS_WARN`），不填时 GetCapabilities 标题为空 |
| `PROJECTION` | [mapwms.cpp:2383](Document/Resources/mapserver-8.6.3/src/mapwms.cpp#L2383) | ⚠️ 条件功能必填： `"every layer MUST have one or produce a warning"` |
| `MAP.extent` | loadMapInternal 解析时检查格式但不强制存在 | ⚠️ 功能必填：无范围无法渲染/定位 |

### C.5 待修正项清单

**从 `business_required` 中移除**（这些字段已有默认值）：

| 对象 | 字段 | 默认值来源 |
|------|------|-----------|
| MAP | `imagetype` | `defaults-override.json`: `"png"` |
| MAP | `status` | `defaults-override.json`: `"on"` |
| LAYER | `connectiontype` | `defaults-override.json`: `"local"` |
| LAYER | `projection` | `defaults-override.json`: `["init=epsg:3857"]` |
| LAYER | `status` | `defaults-override.json`: `"on"` |
| METADATA | `ows_enable_request` | `defaults-override.json`: `"*"` |
| STYLE | `color` | `defaults-override.json` 条件默认值 |
| LABEL | `font`, `size`, `color` | `defaults-override.json` 各默认值 |

**从 `required` 中移除**（这些字段不是语法必填）：

| 对象 | 字段 | 说明 |
|------|------|------|
| LAYER | `name` | MapServer 解析允许无 name 的 LAYER |
| MAP | `name` | MapServer 解析允许无 name 的 MAP（但 OGC 服务需要） |

**新增为 `required_when`**（条件触发时必填）：

| 对象 | 字段 | 条件 | 说明 |
|------|------|------|------|
| METADATA | `ows_title` | `len(session.service_types) > 0` | 启用任何 OGC 服务时需要 |
| METADATA | `ows_onlineresource` | `len(session.service_types) > 0` | 可由 MAP.name 推导，但显式填写更可靠 |
| LAYER | `name` | `len(session.service_types) > 0` | OGC 服务需要图层标识 |
| LAYER | `data` | `connectiontype == 'local'` | 本地文件数据源需要路径 |

**建议新增 `functional_required` 概念**（不填不会报错，但功能不完整，UI 用橙色 `◆` 标记）：

| 对象 | 字段 | 理由 |
|------|------|------|
| MAP | `extent` | 无空间范围无法定位/渲染 |
| MAP | `projection` | 无投影则 OGC 服务无法提供坐标参考系 |
| LAYER | `data` | 无数据源则图层为空 |
