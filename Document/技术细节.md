## 1. 设计前提

### 1.1 模板资源是单一事实源

配置树的结构、参数类型、默认值、枚举范围**全部来自 `mapfile-schema-8-4.json`**。LLM 不感知这些规则，它只负责理解用户自然语言、返回解释建议、返回参数修改值。**规则、校验、类型映射、默认值填充**全部由后端确定性代码处理。

### 1.2 模板资源存放位置

```
SourceCode/data/templates/                          # 模板资源（人工维护，git 跟踪）
├── mapfile-schema-8-4.json                         # MapServer 8.4 JSON Schema（主模板）
├── aliases.json                                    # 业务别名映射
├── required.json                                   # 必填规则
├── phase-map.json                                  # 对象类型 → 阶段归属
├── defaults-override.json                          # 默认值覆盖/补充
├── dependencies.json                               # 跨字段依赖
├── custom-allowed.json                             # 自定义属性白名单
├── service-metadata.json                           # 服务类型参数白名单映射
└── object-fields.json                              # Schema 未覆盖的业务补充字段

SourceCode/data/
└── mapguide_rules.json         # 生成产物，运行时唯一读取的规则文件
```

### 1.3 与 v1 的关键差异

| 方面 | v1 | v2 |
|------|-----|-----|
| 交互入口 | 中间对话面板 | 左侧配置树 |
| LLM 角色 | 主动引导用户填参 | 被动回答用户问题 |
| 配置规则来源 | `mapguide_rules.json` + 人工编码 | 直接从 `mapfile-schema-8-4.json` 映射 |
| 关注点 | 无 | `focus_param`（扁平路径） |
| 会话历史 | 全部保留 | 用户切换关注点后可清空 |
