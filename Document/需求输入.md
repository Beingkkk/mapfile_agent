# MapGuide 需求输入文档

> **定位**：交互式 Mapfile / MapCache XML 配置编辑器，LLM 辅助问答
> **原则**：用户主导配置，LLM 负责解释与纠错
> **版本**：MapServer 8.4（锁定）
> **MVP 范围**：PostGIS / Shapefile → WMS / WFS / WCS / WMTS / TMS → 可选 MapCache 磁盘缓存
> **对应技术文档**：`Document/技术细节.md`

---

## 1. 核心设计原则

### 1.1 用户主导 + LLM 辅助

| 职责 | 用户（通过配置树） | LLM（通过问答面板） | 后端 |
|------|-------------------|-------------------|------|
| 填写参数值 | ✅ 直接编辑 | ❌ | — |
| 增删图层/对象 | ✅ 点击按钮 | ❌ | — |
| 实时校验 | ✅ 失焦触发 | ❌ | ✅ `ValidationPipeline` 执行 |
| 参数含义解释 | ❌ | ✅ 回答用户提问 | — |
| 填写建议/示例 | ❌ | ✅ 回答用户提问 | — |
| 错误原因分析 | ❌ | ✅ 回答用户提问 | — |
| 发布参数引导 | ❌ | ✅ 回答用户提问 | — |
| 别名转换 | — | ❌ | ✅ `TemplateMapper.resolve_alias()` |
| 类型转换 | — | ❌ | ✅ `ValidationPipeline` 类型层 |
| 条件依赖验证 | — | ❌ | ✅ `ValidationPipeline` 语义层 |
| mappyfile 语法校验 | — | ❌ | ✅ `mappyfile.validate(version=8.4)` |
| 默认值填充 | — | ❌ | ✅ `TemplateMapper` + 默认值规则 |
| 推导参数 | — | ❌ | ✅ `dependencies.json` 推导边 |

> **一句话**：用户直接操作配置树填写参数；LLM 只在用户提问时回答；后端负责所有校验和转换。

### 1.2 无主动引导，纯被动响应

- LLM **不主动**问用户问题、不主动引导填参
- LLM 只在两种情况下响应：
  1. 用户主动提问（自由文本或选中属性提问）
  2. 校验错误时，用户点击「发送到 LLM 求助」
- 后端通过 `QAService.answer()` 处理用户提问，单 Prompt 架构不变：`_framework.j2` 约 200 tokens，运行时注入当前 map 结构 + 关注点 + 错误信息
- LLM 返回的 `params_update` 通过 `UpdateResolver` 把 `line` 解析为 `path`，再写入 `ConfigTree`

### 1.3 即时验证、即时反馈

- 每次用户编辑参数失焦后，自动触发后端完整验证闭环
- 校验错误在树上标红，用户可选择发送到 LLM 求助
- 手动校验按钮始终可用

### 2.1 左栏：配置树 ConfigTree

配置树是用户操作的核心区域，直接对应 Mapfile 对象结构。

**服务类型选择**：

配置树顶部（MAP 节点 header 下方）显示服务类型选择栏，用户勾选需要启用的 OGC 服务：

```
┌─ MAP ──────────────────────────┐
│ 服务类型：[x] WMS  [x] WFS  [ ] WCS  [x] MapCache(WMTS/TMS)
│ name: [city_map        ]       │
│ imagetype: [png ▼]             │
│ ...
```

- **WMS**、**WFS**、**WCS**：在 WEB.METADATA 中启用对应元数据参数
- **MapCache**：在根节点下显示 `CACHE` 对象，用于配置 WMTS/TMS 瓦片缓存
- 可多选，MapServer 一份 `.map` 可同时支持多种服务
- 切换服务类型后，METADATA 中可见参数自动过滤，已填写的隐藏参数值**保留**（取消勾选后再勾选可恢复）

**树结构层次**：
```
MAP
├── name, imagetype, projection, status, extent, units, ...
├── 服务类型选择栏 [x] WMS [x] WFS [ ] WCS [x] MapCache
├── WEB
│   └── METADATA
│       ├── ows_title, ows_onlineresource          ← 通用（OWS）
│       ├── wms_title, wms_onlineresource, ...     ← 仅 WMS 勾选时显示
│       ├── wfs_title, wfs_onlineresource, ...     ← 仅 WFS 勾选时显示
│       └── wcs_title, wcs_onlineresource, ...     ← 仅 WCS 勾选时显示
├── LAYER[0]
│   ├── name, type, connectiontype, connection, data, projection, status, ...
│   ├── METADATA
│   │   └── gml_include_items, gml_featureid, ...  ← 仅 WFS 勾选时显示
│   ├── CLASS[0]
│   │   └── STYLE[0]
│   │       ├── color, outlinecolor, width, symbol, size, ...
│   │       └── status
│   └── [+] 添加 CLASS
├── LAYER[1]
│   └── ...
├── [+] 添加 LAYER
└── CACHE（当勾选 MapCache 时显示）
    ├── type, base, expiry, grid, ...
    └── status
```

> **WMTS/TMS 说明**：WMTS 和 TMS 不直接通过 Mapfile 配置，而是通过 MapCache（`mapcache.xml`）实现。用户在配置树中勾选 MapCache 后，CACHE 对象展开，配置完成后导出时同时生成 `.map` 和 `mapcache.xml`。

**节点就地编辑控件**（根据 `mapguide_rules.json` 的 `value_type` 自动映射）：

| value_type | 控件 | 示例 |
|-----------|------|------|
| `string` | 文本输入框 | name, connection |
| `enum` | 下拉选择框 | connectiontype ∈ {local, postgis, ogr, wms} |
| `integer` | 数字输入框 | width, size |
| `float` | 数字输入框（允许小数）| opacity |
| `boolean` | 开关/复选框 | debug, dump |
| `color` | 颜色选择器 | color, outlinecolor |
| `array` | 标签输入框（逗号分隔）| projection = ["init=epsg:3857"] |
| `expression` | 文本输入框（语法高亮）| data = "geom from buildings" |

> **前端实现方案**：采用**分层渲染**，不使用 Naive UI 的 `n-tree` 组件。
>
> - **对象节点**（MAP / LAYER / CLASS / STYLE）渲染为可展开折叠的 `ObjectCard` 组件，管理结构层级、展开状态、增删操作。
> - **属性叶子**在对象卡片内部以表单列表渲染，通过统一的 `FieldEditor` 组件按 `value_type` 自动分发到 8 种原子编辑器（基于 Naive UI 的 `n-input`/`n-select`/`n-color-picker`/`n-switch` 等）。
> - STYLE 为最底层对象（无子对象），直接在父 CLASS 卡片内联渲染，不做为独立可折叠卡片。

**校验状态可视化**：
- 每个对象节点（MAP / LAYER / CLASS / STYLE / MapCache）右侧显示 `status` 图标
- `✓ pass` — 绿色，该校验通过
- `⚠️ fail` — 红色，该校验有错误，点击展开错误详情
- `○ idle` — 灰色，尚未校验或无需校验
- 错误详情悬浮/展开显示具体错误信息

**对象操作按钮**：
- `[+]` 在同级添加新对象（如添加 LAYER、添加 CLASS）
- `[×]` 删除对象（需确认）
- 对象展开/折叠 `▶` / `▼`
- `➕ 添加自定义属性` — 弹出模态框，定义 key 名称 + 描述 + 类型

**自定义属性**：

Mapfile 支持大量扩展参数，部分节点允许用户添加不在预定义 Schema 中的自定义 key-value。每个对象节点底部提供「➕ 添加自定义属性」按钮：

```
├─ LAYER[0]
│   ├── name: [buildings]
│   ├── ...
│   └── ➕ 添加自定义属性
```

点击后弹出模态框，用户输入：

| 字段 | 必填 | 说明 |
|------|------|------|
| key 名称 | 是 | 属性名，如 `transparency`、`filteritem` |
| 描述 | 否 | 该属性的用途说明，仅用于提示 |
| 类型 | 是 | string / enum / integer / float / boolean / color / array |

添加后，该属性显示在树中，左侧标记为 `✎ 自定义`（蓝色小笔图标），可被编辑、删除。

**参数必填性标识**：

每个属性行左侧显示必填性标记，帮助用户识别优先级：

| 标记 | 含义 | 样式 | 说明 |
|------|------|------|------|
| `*` | **必填** | 红色星号，跟随 key | 必须填写，否则校验无法通过 |
| `D` | **默认** | 灰色小字 superscript | 已填入后端默认值，可修改也可保留 |
| `○` | **可选** | 浅灰色空心圆 | 不填不影响基本功能 |
| `→` | **推导** | 灰色箭头 | 后端自动推导，通常只读 |

示例：
```
* name:     [city_buildings    ]   ← 必填，用户已填
D imagetype: [png ▼]              ← 默认值为 png
○ units:    [meters    ]          ← 可选
→ grid:     [GoogleMapsCompati..]  ← 推导，只读
```

**图例说明栏**：

配置树面板顶部固定显示一行图例说明（字号小，可折叠），帮助新用户理解标记含义：

```
📋 图例：* 必填  ·  D 默认值  ·  ○ 可选  ·  → 推导  ·  ✎ 自定义
```

图例栏默认显示，用户熟悉后可点击收起。收起状态保留一个小图标 `ⓘ`，悬停显示图例。

**显示模式切换**：

配置树顶部提供「显示全部 / 仅必填」切换按钮：

- **显示全部**（默认）：展示所有参数（必填 + 可选 + 默认 + 推导）
- **仅必填**：只展示 `*` 必填参数，隐藏 `○` 可选和已填默认值的 `D` 参数
  - 目的：减少视觉噪音，让用户聚焦在必须填写的内容上
  - 例外：如果某个可选参数已被用户手动修改过，即使切到「仅必填」也继续显示
  - 有错误的参数（无论是否必填）始终显示

> **默认值处理**：已填入默认值（`D` 标记）的字段在「仅必填」模式下**隐藏**，除非用户手动修改过该值。这样用户打开应用后看到的「仅必填」视图是干净的，只有真正需要关注的内容。

切换按钮位置：面板 header 右侧，与「展开/折叠」按钮并列。

**底部操作栏**：
- `[📂 导入 Mapfile]` — 选择 `.map` 文件导入；导入成功等价于完全重置当前会话（以文件内容为起点）
- `[✓ 校验全部]` — 触发完整 ValidationPipeline 校验
- `[📤 导出]` — 仅在校验全部通过时可用
- `[🔄 重置 / 新任务]` — 确认后清空所有参数和问答历史
- 校验状态总览：`idle` / `checking` / `pass` / `fail`

### 2.2 右栏：问答面板 QAPanel

问答面板是用户与 LLM 交互的唯一入口。

**两种提问模式**：

| 模式 | 触发方式 | Prompt 注入内容 |
|------|---------|----------------|
| **自由提问** | 不选中任何属性，直接输入 | 当前完整 map 结构 + 校验错误 |
| **属性提问** | 先点击选中树中某个属性/对象，再输入 | 当前完整 map 结构 + **关注点（focus_param）**+ 校验错误 |

**关注点机制**：

用户可以在配置树中选中**属性节点**或**对象节点**作为提问焦点：

| 选中类型 | 视觉反馈 | Prompt 注入 |
|---------|---------|------------|
| **属性节点** | 属性行高亮（左侧蓝色竖条） | `focus_param: "LAYER[0].connectiontype"` |
| **对象节点** | 对象 header 高亮（蓝色背景） | `focus_param: "LAYER[0]"` |

**关注点定位（扁平路径）**：

配置树中所有节点通过**扁平路径**标识，如 `LAYER[0].connectiontype`，该路径同时也是后端 `ConfigTree.get_node(path)` 的查询键。

当选中一个对象节点时，Prompt 中注入该对象的 `focus_param` 路径和包含的所有子字段摘要：

```
focus_param: "LAYER[0]"
```

LLM 回答时可直接引用路径：「LAYER[0].connectiontype 需要配合 LAYER[0].connection 一起使用」。路径是稳定标识符，不受树结构变化影响。

- 点击空白处或按 Esc 取消关注

**消息类型**：
- `user` — 用户提问（白色气泡右侧）
- `bot` — LLM 回答（灰色气泡左侧）
- `system` — 校验错误求助（黄色警告气泡）

**QA 轮数指示器**：

QAPanel 顶部显示当前关注点的问答轮数（如 "💬 3 轮对话"），切换关注点时轮数重置为 0。轮数为 0 时显示 "💬 新话题"。

这个指示器帮助用户理解：切换关注点会清空当前问答历史（保留初始意图），之前的讨论不再影响 LLM 的回答上下文。

**快捷操作**：
- 校验出错的属性旁显示 `?` 按钮，点击自动发送「这个参数为什么报错？」到 LLM
- LLM 回复中包含代码示例时，自动渲染代码块（带复制按钮）

## 3. 交互流程

### 3.1 典型场景 A：从零开始配置 PostGIS → 多服务发布

```
用户打开应用
    │
    ▼
配置树显示 MAP 根节点（默认展开），服务类型选择栏默认勾选 [x] WMS
    │
    ▼
可选：从 data/init_session_intent.json 读取默认意图（不强求用户输入）
    │
    ▼
用户在服务类型栏勾选 [x] WMS 和 [x] WFS
  → METADATA 中自动显示 wms_* 和 wfs_* 相关参数
  → 通用 ows_title 默认显示，wms_title / wfs_title 为可选（填写专用前缀则覆盖 ows_）
    │
    ▼
用户在 MAP.name 输入 "city_buildings"
  → 失焦触发实时校验 → `ValidationPipeline.validate_field()` 执行 alias/type/semantic
  → 其他字段标为"待填写"或默认值（如 imagetype=png）
    │
    ▼
用户点击 [+] 添加 LAYER
  → `ConfigTree.add_object()` 生成 LAYER[0]
  → 默认填入 connectiontype=local, projection=["init=epsg:3857"]
    │
    ▼
用户将 connectiontype 改为 "postgis"（下拉选择）
  → 失焦触发校验 → `ValidationPipeline` 检测依赖：postgis 要求 connection+data 必填
  → connection 和 data 字段标红，显示"条件必填"
    │
    ▼
用户勾选 [x] MapCache（WMTS/TMS）
  → 根节点下方显示 CACHE 对象
  → 默认填入 grid=GoogleMapsCompatible, type=disk, base=/tmp/cache
    │
    ▼
用户点击 connection 旁的 ? 按钮
  → 右侧面板自动发送 question 消息，focus_param="layers.0.connection"
  → `QAService.answer()` 调用 LLM
  → LLM 回复：「当 connectiontype=postgis 时，connection 是必填字段，格式为 host=... dbname=...」
    │
    ▼
用户在 connection 填入 "host=localhost dbname=city user=postgres"
  → 失焦校验 → 通过
    │
    ▼
用户继续填写其他参数...（同上模式）
    │
    ▼
所有必填参数填完后，用户点击 [✓ 校验全部]
  → `ValidationPipeline.validate_tree()` 执行四层完整验证
  → 通过 → 导出按钮激活
    │
    ▼
用户点击 [📤 导出]
  → 确认弹窗显示：
    "将导出以下文件：
      · city_buildings.map（含 WMS + WFS 配置）
      · mapcache.xml（WMTS/TMS）"
  → 确认后生成 .map + mapcache.xml
```

### 3.2 典型场景 B：询问参数含义

```
用户选中 LAYER[0].projection
    │
    ▼
右侧面板显示「当前关注：LAYER[0].projection」
    │
    ▼
用户输入："这个参数是什么意思？3857 和 4326 有什么区别？"
    │
    ▼
LLM 回答 projection 的含义，EPSG 代码的区别，以及推荐值
```

### 3.3 典型场景 C：发布建议

```
用户不选中任何属性，直接输入：
"我要把 buildings 表发布成 WMS + WFS，需要配置哪些参数？"
    │
    ▼
LLM 基于当前 map 结构和服务类型选择，列出需要填写的参数清单：
  · WMS：wms_title, wms_onlineresource, wms_enable_request, ...
  · WFS：wfs_title, wfs_onlineresource, wfs_enable_request, gml_include_items, ...
  · LAYER 级别：name, type, projection, connection, data
  · 通用：可使用 ows_title 替代各服务的独立 title（推荐简化配置）
    │
    ▼
用户点击 LLM 建议中的参数 → 自动定位到对应节点并高亮
```

### 3.4 典型场景 D：修复校验错误

```
用户填了一个错误值：color = "蓝色"
    │
    ▼
失焦触发校验 → 类型错误：color 应为 RGB 数组 [R,G,B]
    │
    ▼
该字段标红，显示错误信息
    │
    ▼
用户点击字段旁的 ? 按钮
    │
    ▼
右侧面板发送：「LAYER[0].CLASS[0].STYLE[0].color = "蓝色" 为什么报错？」
    │
    ▼
LLM 回答："color 必须是 RGB 数组格式，如 [0, 0, 255] 表示蓝色。
你输入的是字符串"蓝色"，请改为 [0, 0, 255]。"
```

### 3.5 典型场景 E：导入已有 Mapfile

```
用户点击底部 [📂 导入 Mapfile]
    │
    ▼
Electron dialog.showOpenDialog 弹出文件选择器（过滤 *.map）
    │
    ▼
用户选择文件 → Electron 主进程读取内容
    │
    ▼
通过 IPC → 渲染进程 → WS: import_mapfile { content }
    │
    ▼
后端：mappyfile.loads(content) 尝试解析
    │
    ├──▶ 解析失败 → WS: import_result { success: false, error: "Mapfile 语法错误: ..." }
    │              → 前端弹窗提示，当前会话不受影响
    │
    └──▶ 解析成功
              │
              ▼
         销毁旧 ConfigSession，以解析后的 dict 为 params 新建 session
              │
              ▼
         ConfigTree._build_tree() — 递归构建树
              │  · Schema 中已定义的字段 → 正常 TreeLeaf
              · 不在 Schema 中的字段 → 全部标记为 custom=True（✎ 自定义）
              ▼
         ValidationPipeline.validate_tree() — 四层完整校验
              │
              ▼
         清空问答历史（保留 intent_message）
              │
              ▼
         WS: tree_state { params_snapshot, validation_state, errors }
              │
              ▼
         前端：完全刷新配置树，显示校验结果，轮数归零
```

**分层策略说明**（导入失败的处理）：

| 阶段 | 判定标准 | 行为 |
|------|----------|------|
| **mappyfile 解析失败** | `mappyfile.loads()` 抛出异常 | **拒绝导入**，前端弹窗显示具体语法错误；当前会话不受影响 |
| **解析成功但业务校验失败** | 四层校验中 L1-L3 或 L4 有错误 | **允许导入**，错误标红显示在树上；用户可在应用内修复或问 LLM |
| **解析成功且校验通过** | 四层校验全部通过 | **正常导入**，导出按钮直接可用 |

**自定义属性识别**：导入时遇到 schema 未定义的字段，统一标记 `custom=True`，左侧显示 `✎` 图标。不额外区分"合法扩展"与"拼写错误"——用户可自行判断删除或保留。

**会话状态**：导入 = 完全重置。旧 session 销毁，问答历史清空（`intent_message` 保留），轮数归零，focus_param 置空。

## 4. 校验机制

### 4.1 触发时机

| 触发方式 | 执行范围 | 说明 |
|---------|---------|------|
| **实时校验** | 单字段 | 字段失焦时，校验该字段的类型、必填、依赖 |
| **对象校验** | 对象内所有字段 | 对象展开/折叠时，或对象内任一字段变更后延迟 500ms |
| **手动校验** | 完整 Mapfile | 点击 [✓ 校验全部]，执行四层完整验证 |
| **导出前校验** | 完整 Mapfile | 点击 [📤 导出] 前自动执行，不通过则阻断 |
| **导入后校验** | 完整 Mapfile | 导入成功后自动执行四层完整验证，结果随 tree_state 返回 |

### 4.2 四层验证

| 层级 | 执行者 | 校验内容 | 失败处理 |
|------|--------|----------|----------|
| **别名解析** | `ValidationPipeline` 第 1 层 | 调用 `TemplateMapper.resolve_alias()` 转换别名 | 转换成功后静默更新 |
| **类型转换** | `ValidationPipeline` 第 2 层 + Pydantic | 类型、枚举、范围 | 字段标红 + 错误提示 |
| **语义校验** | `ValidationPipeline` 第 3 层 | 读取 `dependencies.json` 检查条件必填、互斥、依赖 | 字段标红 + 错误提示 |
| **Mapfile 语法** | `mappyfile.validate(version=8.4)` | 完整 Mapfile 结构合法性 | 全局错误面板提示 |

**执行范围差异**：
- 字段失焦：只执行前 3 层（不完整 map 无法做 mappyfile 语法校验）
- 添加/删除节点、手动校验、导出：执行全部 4 层

### 4.3 错误展示

- **字段级错误**：字段边框变红，下方悬浮/展开显示错误信息，点击 ? 发送给 LLM
- **对象级错误**：对象节点 status 变 ⚠️，展开显示该对象内所有错误
- **全局错误**：校验面板底部显示完整错误列表

### 4.4 LLM 容错策略

LLM 输出不稳定性采用**宽容降级**策略：

- LLM 返回的 `params_update` 如有格式错误，**静默忽略该项更新**，继续处理其他更新和回答文本
- JSON 解析完全失败时，降级为纯文本回答展示给用户，不影响配置树状态
- 错误提示使用封装语气（如"抱歉，我理解您的意图有些困难，能否换个方式描述？"），不裸抛技术异常

## 5. 导出逻辑

### 5.1 导出条件

- `[📤 导出]` 按钮仅在**全部四层校验通过**时可用
- 不可跳级导出：有错误 → 按钮禁用 → 提示"请先修复校验错误"

### 5.2 导出内容

| 文件 | 条件 | 说明 |
|------|------|------|
| `{map_name}.map` | **始终导出** | 一份文件包含所有已勾选服务的配置（WMS + WFS + WCS） |
| `mapcache.xml` | 勾选 MapCache 时 | 包含 WMTS/TMS 缓存配置 |

### 5.3 导出流程

1. 用户点击 [📤 导出]
2. 前端自动触发完整校验（复用 validate 逻辑）
3. 校验通过 → 弹出确认对话框：
   · 显示将要导出的文件列表
   · **显示已启用的服务类型摘要**（如"含 WMS + WFS + MapCache(WMTS/TMS)"）
4. 用户确认 → 后端生成文件内容 → 通过 WS 返回 export_result（base64）
5. 前端调用 Electron `dialog.showSaveDialog` 选择保存目录
6. Electron 主进程将文件写入用户选择的目录
7. 校验失败 → 导出按钮禁用，显示错误列表

## 7. 重置与导入行为

### 7.1 重置

- **"🔄 重置 / 新任务"** 按钮固定在配置树底部
- 点击后弹出确认对话框，确认则：
  - 清空所有参数（恢复为仅含 MAP 根节点的空树）
  - 清空问答历史（保留 `intent_message`）
  - 状态完全恢复为初始态
- **无历史保存**：不保留任何之前的配置或对话

### 7.2 导入

- **"📂 导入 Mapfile"** 按钮固定在配置树底部（与重置/校验/导出并列）
- 点击后 Electron 弹出文件选择器，过滤 `*.map` 文件
- 导入流程等价于**"重置 + 从文件填充参数"**：
  - 销毁旧 session，以文件解析结果新建 session
  - 清空问答历史（保留 `intent_message`）
  - 自动执行四层完整校验，结果标在树上
- **解析失败保护**：`mappyfile.loads()` 失败时，当前会话不受影响

## 9. 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 用户不熟悉 Mapfile 结构 | LLM 问答面板随时可问；参数徽章提示分类 |
| LLM 回答不准确 | 后端强验证兜底；用户可直接修改树值覆盖 |
| 实时校验性能问题 | 字段级校验只做 alias+type+semantic，mappyfile 校验仅在手动/导出时执行 |
| 树结构复杂难导航 | 支持展开/折叠；搜索过滤；对象跳转 |

## 附录 A：服务类型快速参考

| 服务 | 配置位置 | 关键参数 | 数据源要求 |
|------|---------|---------|-----------|
| **WMS** | MAP.WEB.METADATA | `wms_title`, `wms_onlineresource`, `wms_enable_request`, `wms_srs` | `LAYER.type` ∈ {point, line, polygon, raster} |
| **WFS** | MAP.WEB.METADATA | `wfs_title`, `wfs_onlineresource`, `wfs_enable_request`, `wfs_srs`, `gml_include_items` | `LAYER.type` ≠ raster |
| **WCS** | MAP.WEB.METADATA | `wcs_title`, `wcs_onlineresource`, `wcs_enable_request`, `wcs_srs` | `LAYER.type` = raster |
| **WMTS** | MapCache (`mapcache.xml`) | `<service type="wmts">` | 底层回源到 MapServer WMS |
| **TMS** | MapCache (`mapcache.xml`) | `<service type="tms">` | 底层回源到 MapServer WMS |

> **通用前缀**：MapServer 支持 `ows_*` 前缀作为 WMS/WFS/WCS 的公共前缀。如果未设置 `wms_title`，则回退到 `ows_title`。推荐优先使用 `ows_*` 简化配置，仅在需要差异化时使用服务专用前缀。
